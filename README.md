# 自动投递助手（无 AI）

这是一个面向招聘网站网页端的 Tampermonkey/油猴脚本集合，当前包含 BOSS 直聘、猎聘、智联招聘三个独立脚本。

> 脚本不包含 AI 功能，不连接第三方服务，不设置自动更新地址。所有投递、扫描、过滤和历史记录都在当前浏览器页面与本地 `localStorage` 中完成。

## 安装

1. 在 Chrome、Edge 或其他 Chromium 浏览器中安装 Tampermonkey 或同类用户脚本管理器。
2. 打开 Tampermonkey 管理面板，点击“添加新脚本”。
3. 将需要的平台脚本完整复制进去并保存，例如：
  - BOSS：复制 `boss-auto-apply.user.js`
  - 猎聘：复制 `liepin-auto-apply.user.js`
  - 智联：复制 `zhilian-auto-apply.user.js`
4. 打开对应招聘网站并登录账号。
5. 页面右上角会出现脚本面板，确认筛选条件后再点击“开始”。

建议三个平台脚本分开安装、分开启用。不要把多个脚本内容粘在同一个 Tampermonkey 脚本里。

## 使用流程

1. 先在招聘网站页面中手动设置城市、关键词、薪资、经验等筛选条件。
2. 等职位列表加载完成后，检查脚本面板中的配置。
3. 设置最大投递数和间隔秒数。
4. 按需要开启外企过滤、外包拦截、活跃度过滤、历史跳过等选项。
5. 点击“开始”或“开始投递”。
6. 脚本会按当前页面职位列表逐个处理，达到上限、没有更多职位或遇到异常时停止。

## 通用功能

- **自动投递**：扫描当前页面职位卡片，找到可投递/沟通/申请按钮后自动点击。
- **最大投递数**：达到设置的成功投递数量后自动停止。
- **投递间隔**：每次处理后等待指定秒数，并带有轻微随机延迟，减少过快操作。
- **自动加载更多**：列表没有待处理职位时，尝试点击下一页、加载更多或滚动页面。
- **历史跳过**：已投递、已沟通或本地记录过的职位会跳过，避免重复操作。
- **外企过滤**：开启后只处理命中外企关键词的职位。
- **外包拦截**：开启后跳过命中外包公司名或外包关键词的职位。
- **活跃度过滤**：开启后只处理近期活跃的 Boss/HR，无法识别活跃度时会跳过。
- **本地保存**：配置和历史记录保存在浏览器 `localStorage` 中。
- **日志面板**：面板中会显示扫描、跳过、成功、失败和停止原因。

## 平台差异

### BOSS 直聘

- 使用 BOSS 页面内接口发起沟通/投递。
- 支持投递成功后按岗位生成招呼语并发送。
- 会监听职位列表接口响应，结合 DOM 扫描补充职位数据。
- 会检查已沟通状态，尽量避免重复打招呼。
- 支持过滤不活跃 Boss。

### 猎聘

- 主要通过页面按钮点击完成投递、应聘或开聊。
- 会监听猎聘开聊相关接口，用于判断成功、失败或风控。
- 支持页面确认弹窗、安全提示弹窗处理。
- 支持列表页下一页、URL 翻页和滚动加载。
- 支持过滤不活跃 HR。

### 智联招聘

- 主要通过页面按钮点击完成投递/申请。
- 会处理“确认投递”“继续投递”“确定”等确认弹窗。
- 会监听投递相关网络请求，用于判断成功、失败或异常。
- 为避免刷新循环，智联脚本不会自动启动，也不会通过修改 URL 强制翻页。
- 打开页面后只加载面板和扫描职位，必须手动点击“开始”才会投递。

## 配置说明

### 最大投递数

控制本轮最多成功投递多少个职位。达到数量后脚本自动停止。

### 间隔秒数

每次投递或跳过后的等待时间。建议不要设置太低，避免触发平台频控或风控。

### 自动下滑/翻页

开启后，当当前列表没有待处理职位时，脚本会尝试点击页面上的“下一页”“加载更多”“换一批”等按钮，或向下滚动加载更多职位。

智联版为了避免刷新循环，禁用了通过修改 `location.href` 强制翻页，只做页面内按钮点击和滚动。

### 跳过历史投递

开启后，脚本会根据职位 ID、职位链接、公司名和职位名生成本地历史记录。后续再次遇到同一职位时会跳过。

历史记录默认保留约 180 天，最多保存 3000 条。

### 只投外企

开启后，只有职位、公司或原始数据中命中外企关键词时才会投递。

默认关键词包含外企、欧美、美国、欧洲、德国、法国、英国、瑞士、瑞典、荷兰、丹麦、芬兰、挪威、日本、韩国、新加坡、跨国公司、世界 500 强、Fortune、NASDAQ、NYSE、外资、合资、独资、APAC、MNC、Global、International 等。

### 外包拦截

开启后，命中外包公司名单或外包相关关键词的职位会被跳过。

默认名单包含中软国际、中科创达、博彦科技、软通动力、文思海辉、东软、法本信息、柯莱特、中科软、亚信科技、外企德科、FESCO Adecco、埃森哲、Accenture、凯捷、Capgemini、驻场、外派、劳务派遣、OD、ODC、ITO、BPO 等。

### 活跃度过滤

开启后，脚本会根据页面文本、职位卡片、详情信息或接口数据识别 Boss/HR 最近是否活跃。

如果无法识别活跃度，脚本会按“不活跃/未知”处理并跳过。

## 本地数据

三个脚本使用相互独立的本地存储 key：

- BOSS：`boss_auto_apply_settings_v1`、`boss_auto_apply_contact_history_v1`
- 猎聘：`liepin_auto_apply_settings_v1`、`liepin_auto_apply_history_v1`
- 智联：`zhilian_auto_apply_settings_v1`、`zhilian_auto_apply_history_v1`

如果需要重置配置或历史，可以：

- 使用脚本面板中的“清空历史”按钮。
- 或在浏览器开发者工具中手动清理对应 `localStorage` key。

## 注意事项

- 使用前请先确认账号已登录、简历完整、求职状态正常。
- 建议先把最大投递数设置为较小值，例如 3 到 5，确认页面适配正常后再提高。
- 不建议把间隔秒数设置得过低，避免触发平台频控、验证码或风控。
- 如果页面结构更新，脚本可能找不到职位卡片或投递按钮，需要更新选择器。
- 如果出现验证码、登录失效、风控提示或投递上限，脚本会记录日志并停止或跳过。
- 自动投递可能影响账号行为评分，请自行控制使用频率和投递范围。

## 更新脚本

脚本没有自动更新地址。更新方式：

1. 打开本仓库中的对应 `.user.js` 文件。
2. 复制完整内容。
3. 在 Tampermonkey 中打开旧脚本，替换全部内容并保存。
4. 刷新招聘网站页面。
